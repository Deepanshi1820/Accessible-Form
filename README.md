# Accessible Form
Accessible Forms are created to serve the purpose of inclusitivity, means everyone including those
with disabilities (visual, motor, cognitive) or using assistive tech like screen readers or voice 
commands, to easily understand, navigate, and complete online forms.

Tech Stack Used: HTML & CSS

## Key Insights:
+ Each Input field is associated with ```aria-required``` (to specify the required input field) and ```autocomplete``` (to describe what the input field is about)
+ Each Span element is assosicated with ```aria-label``` for span class description
+ ARIA Attributes: Included to coordinate with screen-readers
+ COLOR CONTRAST: WCAG (Web Content Accessibility Guidelines) standards, making the form readable for users with visual impairments.
+ FOCUS STATE: Style the focus state of each input field so that users navigating with a keyboard can easily see which field is currently active.
+ ERROR MESSAGING: Error messages that can be displayed when a user inputs invalid data.
